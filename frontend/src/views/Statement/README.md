# Модуль Statement — Страница инвентаризационной ведомости

## Обзор

Модуль `Statement` — страница для работы с инвентаризационной ведомостью, загруженной из почтового вложения. Отображает сгруппированные строки ведомости с цветовой индикацией, позволяет фильтровать записи, управлять актуальностью, создавать объекты через QR-сканирование и просматривать существующие объекты.

## Структура компонентов

### Основные компоненты

1. **StatementPage.vue** — корневой компонент, оркестрирует загрузку, фильтрацию и модалки
2. **StatementTable.vue** — таблица ведомости с группировкой, цветами и чекбоксами актуальности

### Модальные окна

3. **ObjectViewModal.vue** — просмотр объектов и записей по инвентарному номеру и партии
4. **ObjectFormModal.vue** — создание/редактирование объекта (из модуля ObjectForm)
5. **UniversalFilterModal.vue** — универсальная модалка фильтрации (из `components/common`)

### Композаблы

- **useStatementData.js** — загрузка сырых данных ведомости, SSE-обновления, офлайн-поддержка
- **useStatementAggregation.js** — группировка строк, расчёт цветов и количества
- **useTableFilter.js** — логика фильтрации агрегированных данных
- **useActualManager.js** — управление флагом `isActual`

## Поток данных

1. **Загрузка**: `useStatementData` получает сырые строки ведомости с сервера или из IndexedDB
2. **Агрегация**: `useStatementAggregation` группирует строки:
   - Активные (`isActual = true`): группировка по `invNumber + partyNumber`
   - Неактивные (`isActual = false`): группировка только по `invNumber`
3. **Цвета групп**:
   - Красный (1) — все строки без объекта
   - Жёлтый (2) — часть строк с объектом, часть без
   - Зелёный (3) — все строки с объектом
   - Серый (4) — неактуальные (снят флаг `isActual`)
4. **Сортировка**: красные → жёлтые → зелёные → серые, внутри группы по алфавиту
5. **Фильтрация**: `useTableFilter` фильтрует агрегированные строки по `invNumber` и `buhName`
6. **Отображение**: `StatementTable` отрисовывает отфильтрованные данные

## Функциональность

### 1. Просмотр ведомости

- Таблица с агрегированными строками и цветовой индикацией
- Отображение инвентарного номера, партии (для ОСВ), количества и наименования
- Клик по строке открывает `ObjectViewModal` для просмотра объектов и записей

### 2. Фильтрация

- Фильтр по инвентарному номеру (`invNumber`)
- Фильтр по наименованию (`buhName`)
- Модалка `UniversalFilterModal` с поиском и множественным выбором
- Активные фильтры подсвечивают заголовки колонок жёлтым
- Кнопка «Сбросить фильтры» появляется при активных фильтрах

### 3. Управление актуальностью

- Чекбокс в каждой строке управляет флагом `isActual`
- Снятие флага переводит строку в неактуальные (серая группа, группировка без партии)
- Изменение применяется ко всем записям с данным `invNumber` в рамках ведомости

### 4. Создание объектов

- Через `ObjectViewModal`: QR-сканирование для записей без объекта
- Через `StatementTable`: событие `@qr-scan` напрямую открывает `ObjectFormModal`
- После создания объекта флаг `haveObject` обновляется, данные перезагружаются

### 5. Просмотр объектов

- `ObjectViewModal` показывает два списка:
  - Записи ведомости с `haveObject = false` (с кнопкой QR-сканирования)
  - Существующие объекты (с кнопкой «Открыть»)
- Фильтрация по `invNumber + partyNumber` для точного совпадения

### 6. Офлайн-режим

- Данные ведомости загружаются из IndexedDB
- Создание объектов работает офлайн (сохраняются в кэш)
- SSE-события не подключаются в офлайне

## SSE-события

| Событие | Отправитель | Действие |
|---------|-------------|----------|
| `statement-updated` | Бэкенд | Перезагрузить ведомость (если тот же `attachmentId`) |
| `statement-active-changed` | Бэкенд | Перенаправить на `/` (если другая ведомость стала активной) |
| `statement-deleted` | Бэкенд | Перенаправить на `/` (если текущая ведомость удалена) |

## Интеграция с API

### Ведомость
- `GET /api/statements/:attachmentId` — загрузка ведомости
- `POST /api/statements/update-actual` — изменение `isActual`
- `POST /api/statements/update-have-object` — установка `haveObject = true`
- `GET /api/statements/by-inv?inv=&party=&zavod=&sklad=` — поиск записей без объекта

### Объекты
- `GET /api/objects/by-inv?inv=&party=&zavod=&sklad=` — поиск существующих объектов
- `POST /api/objects` — создание объекта
- `PATCH /api/objects/:id` — обновление объекта
- `GET /api/objects/place-combinations` — варианты местоположений

## Сервисы фронта

- `statement.service.js` — онлайн/офлайн работа с ведомостью
- `object.service.js` — онлайн/офлайн работа с объектами
- `offline-cache-service.js` — кэширование в IndexedDB

## Стили

- Tailwind CSS на всех компонентах
- StatementPage.css — удалён, все стили перенесены в утилитарные классы
- Цветовая схема групп: красный/жёлтый/зелёный/серый через Tailwind-классы в StatementTable
- Кнопка «Сбросить фильтры»: `bg-amber-50 border-amber-400 text-amber-700`

## Меры безопасности

- JWT-авторизация на всех эндпоинтах
- Проверка прав доступа к ведомости через `mol_access` на бэке
- Фронтовые проверки ролей через композабл `useCurrentUser`
