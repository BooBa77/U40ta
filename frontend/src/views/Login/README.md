# Модуль Login - Страница авторизации

## Обзор

Модуль `Login` — страница входа в приложение U40TA. Предоставляет два способа авторизации: через Telegram (временно заблокирован) и через email с подтверждением по коду. После успешной авторизации пользователь получает JWT-токен и перенаправляется на главную страницу.

## Структура компонентов

### Основные компоненты

1. **Login.vue** — корневой компонент страницы входа
2. **EmailLoginModal.vue** — модалка для входа по email с подтверждением кода

### Вспомогательные компоненты

- **PWAInstallButton.vue** — кнопка установки PWA (из `components/ui`)

## Способы авторизации

### 1. Telegram (временно недоступен)

- Авторизация через Telegram Widget
- Заблокирован правительством РФ
- На странице отображается информационное сообщение
- Код закомментирован, легко восстанавливается

### 2. Email (основной способ)

**Процесс авторизации:**

1. Пользователь вводит логин и выбирает домен
2. На email отправляется 4-значный код подтверждения
3. Пользователь вводит код
4. При успешной проверке генерируется JWT-токен
5. Происходит редирект на главную страницу

**Особенности:**

- Корпоративный домен по умолчанию: `@irkutsk-dobycha.gazprom.ru`
- Доступные домены: `irkutsk-dobycha.gazprom.ru`, `mail.ru`, `gmail.com`, `yandex.ru`
- Автоматический парсинг email для создания пользователя:
  - `first_name` = последние 2 символа локальной части
  - `last_name` = всё до последних 2 символов
  - `abr` = первая буква last_name + first_name
- Код действителен 5 минут
- Повторная отправка доступна только после истечения кода
- Восстановление состояния после F5 (сохранение email в localStorage)

## EmailLoginModal.vue

### Состояния

| Состояние | Описание |
|-----------|----------|
| Ввод email | Поле логина + выпадающий список доменов |
| Ожидание кода | Поле для ввода 4-значного кода, таймер 5 минут |
| Повторная отправка | Кнопка "Запросить код подтверждения" после истечения таймера |

### Валидация

**Локальная часть email:**
- Только латиница, цифры, точка, дефис, нижнее подчеркивание
- Минимум 1 символ
- Автоматическая проверка при вводе

**Код подтверждения:**
- Только 4 цифры
- Автоматическое удаление нецифровых символов
- Проверка при вводе

### Восстановление состояния

При обновлении страницы (F5):
1. Проверяется наличие сохраненного email в localStorage (`u40ta_email_login`)
2. Отправляется запрос на проверку статуса кода
3. Если код активен — восстанавливается таймер и состояние
4. Если код истек — состояние сбрасывается

## Интеграция с API

### Эндпоинты авторизации

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/api/auth/email/send-code` | Отправить код на email |
| POST | `/api/auth/email/verify` | Проверить код и получить JWT |
| POST | `/api/auth/email/check-status` | Проверить статус кода |
| POST | `/api/auth/telegram` | Авторизация через Telegram (заблокирован) |

### Форматы запросов

**send-code:**
json
{
  "email": "tregubovsy@irkutsk-dobycha.gazprom.ru"
}

**verify:**
json
{
  "email": "tregubovsy@irkutsk-dobycha.gazprom.ru",
  "code": "1234"
}

**check-status:**
json
{
  "email": "tregubovsy@irkutsk-dobycha.gazprom.ru"
}

### Форматы ответов
**send-code:**
json
{
  "success": true,
  "message": "Код подтверждения отправлен на вашу почту"
}

**verify:**
json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

**check-status:**
json
{
  "hasActiveCode": true,
  "remainingSeconds": 245
}
